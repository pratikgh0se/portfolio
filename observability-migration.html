Observability System Migration & Optimization

Company: Highspot

Dates: April 2024 - Present

Led a monumental initiative to migrate Highspot's legacy observability systems from GCP/GKE to AWS and optimize metric collection, logging, and alerting. This project resulted in significant cost savings, improved system health visibility, and streamlined incident management.

Key Technologies & Tools

Prometheus, Grafana, New Relic, OpenTelemetry, Fluent Bit, Fluentd, Thanos, Elasticsearch, Rollbar, AWS (EKS, Lambda, Firehose, CloudWatch, S3), GCP (GKE), Terraform, Go, Python, Bash, Google App Scripts, OpsGenie, Jira, GitHub Actions.

Situation

Highspot's observability infrastructure was a complex, fragmented landscape spanning multiple legacy tools (Prometheus v2, Thanos for long-term storage, self-hosted Grafana instances, Elasticsearch for logs, Rollbar for error tracking) and cloud providers (GCP/GKE and AWS). This multi-vendor, multi-cloud sprawl led to exorbitant operational costs (daily metric ingestion ~20TB, monthly storage ~256TB), inconsistent monitoring dashboards, severe alert fatigue due to duplicate or unactionable alerts, and prolonged Mean Time To Resolution (MTTR) for incidents because critical data was scattered across disparate systems, making debugging a "war room" scenario.

Task

Consolidate and optimize Highspot's observability stack by executing a comprehensive migration to a unified New Relic platform. This involved deprecating all legacy systems and implementing robust, cost-efficient metric and log management strategies to drastically reduce operational costs, enhance system reliability, and significantly improve incident response capabilities.

Action

  • Conducted a detailed, data-driven analysis of existing observability costs and usage patterns, developing comprehensive deprecation plans for redundant GCP/GKE resources, Prometheus v2 instances, and self-hosted Grafana instances.
  • Spearheaded the complete, phased migration of all observability tools and data streams to a unified New Relic platform.
  • Led the architectural design and full lifecycle deployment of Prometheus-Operator across all production environments, implementing over 200+ recording rules for 1000+ high-cardinality metrics, optimizing scrape intervals, and applying extensive relabel_configs.
  • Debugged and resolved persistent remote write failures and buffer issues with Prometheus sending data to New Relic, collaborating closely with New Relic support.
  • Reconfigured Fluent Bit (for Kubernetes pods) and Fluentd (for EC2 instances) to directly forward logs to New Relic, eliminating intermediate aggregators.
  • Implemented distributed tracing and spans with the New Relic Java Agent in Magma, conducting POCs for custom tracing and debugging uninstrumented time.
  • Orchestrated regional AWS CloudWatch metrics and log streaming to New Relic using Terraform, AWS Kinesis Firehose, and AWS Lambda, ensuring data locality and compliance.
  • Developed Golang scripts and Google App Scripts (.gs) to automate the analysis and identification of high-cardinality and unused metrics.
  • Proposed and successfully implemented a "Crew Ownership and Metrics Budgeting" model, assigning clear ownership of metrics to specific engineering teams.
  • Designed a sophisticated metrics aggregation solution using a custom Kubernetes controller and CRDs.
  • Established a robust, version-controlled alert creation and governance process by integrating OpsGenie and Jira with Terraform scripts, utilizing GitHub Actions for automated validation.
  • Automated alert validation processes across New Relic, Slack, and PagerDuty.
  • Automated observability workflows using Terraform for critical tasks like alert creation, resource cleanup, and Kubernetes cluster backups.
  • Authored comprehensive internal playbooks for tool deprecations and migrations, created detailed design documents, and conducted extensive training sessions.
  • Mentored teammates (Pankaj, Krishna, Venu) on complex Terraform and Kustomize configurations.

Result

  • Estimated annual savings of over $720K by successfully deprecating legacy components and optimizing data pipelines.
  • Daily metric ingestion drastically reduced by 85%+ (from 20TB to 2TB).
  • Monthly storage costs cut by 38% (from 256TB to 160TB).
  • Unique metrics decreased by 33% (8.8K to 5.9K).
  • Unified New Relic platform provided enhanced system health visibility, significantly reduced alert fatigue, and improved MTTR for incidents.
  • Operational efficiency boosted significantly through extensive automation of observability workflows.

Review

This was a highly complex, multi-faceted, and ultimately monumental transformation of Highspot's observability landscape. My leadership in driving this initiative, coupled with deep technical expertise in cloud platforms (AWS, GCP), Kubernetes, and a wide array of observability tools, was absolutely critical to its success. The project delivered massive, quantifiable cost savings while simultaneously improving system reliability, operational agility, and the overall developer experience. The strategic implementation of metric ownership, automated governance processes, and robust data pipelines ensures long-term sustainability and positions Highspot for continuous improvement in its operational capabilities. This project truly showcased my ability to lead large-scale technical migrations with a focus on both immediate impact and long-term strategic value.
