RESTful API Development

Company: Highspot

Dates: April 2024 - Present

Developed secure and scalable RESTful APIs, implementing robust authentication, rate limiting, and API throttling for high availability.

Key Technologies & Tools

Go, Python, Clojure, REST APIs, OpenAPI/Swagger, OAuth2, JWT, API Gateways, Circuit Breakers, Unit Testing, Integration Testing.

Challenge

As Highspot's microservices architecture expanded and the platform grew, there was a continuous demand for new APIs and enhancements to existing ones to support new features, integrations with partners, and internal service-to-service communication. These APIs needed to meet stringent requirements for security, maintain high performance under varying loads, and exhibit resilience against potential abuse or overload, all while ensuring high availability.

Action

  • Meticulously designed clear, consistent, and well-documented API contracts using OpenAPI/Swagger, defining precise endpoints, HTTP methods, request/response formats, data models, and error schemas.
  • Implemented the API logic using a polyglot approach, leveraging Go, Python, and Clojure (within the Magma codebase) depending on the service's specific requirements and existing technology stack, adhering strictly to RESTful principles.
  • Integrated robust authentication mechanisms, including OAuth2 and JWT (JSON Web Tokens), to secure API endpoints, implementing token validation, refresh mechanisms, and secure storage practices.
  • Implemented fine-grained authorization checks (e.g., Role-Based Access Control - RBAC) to ensure that only authenticated and authorized users or services could access specific API resources.
  • To protect backend services from overload and ensure fair usage, implemented comprehensive rate limiting and API throttling strategies at multiple layers (API Gateway and application code), utilizing algorithms like the token bucket or leaky bucket.
  • Designed and implemented consistent and informative error handling strategies across all APIs, providing clear error responses (e.g., standard HTTP status codes, detailed error messages) to consumers.
  • Incorporated critical resilience patterns such as circuit breakers (to prevent cascading failures to downstream services) and retries with exponential backoff for interactions with external dependencies.
  • Developed extensive unit, integration, and end-to-end tests for all API endpoints, covering a wide range of scenarios, including successful requests, various failure conditions, authentication edge cases, and performance under load.
  • Maintained up-to-date, living API documentation that was easily accessible to both internal and external developers.
  • Actively collaborated with frontend developers, mobile teams, and partner integration teams to ensure smooth consumption of the APIs and to address any integration challenges or feedback promptly.

Result

  • Successfully delivered a suite of secure, scalable, and highly available RESTful APIs that effectively met critical business requirements and supported Highspot's expanding platform.
  • The implemented security measures (OAuth2, JWT) and traffic management controls (rate limiting, throttling) protected backend systems from overload, malicious attacks, and resource exhaustion, ensuring high availability and data integrity.
  • This directly contributed to the overall stability, performance, and reliability of Highspot's microservices architecture and enabled faster development cycles for new features.

Review

This project was a critical success, fundamentally transforming Highspot's feature delivery capabilities and accelerating its product development lifecycle. My ability to lead the design, implement complex Clojure code, integrate securely with external services, and conduct thorough, multi-faceted testing was paramount. The collaborative approach, especially in debugging intricate cross-service issues, proved highly effective in overcoming technical hurdles. The system now provides a flexible, resilient, and safe mechanism for future product innovation and experimentation, allowing Highspot to be more agile and responsive to market demands.

RESTful API Implementation (Go)

// Go implementation of secure RESTful API
package main

import (
    "github.com/gin-gonic/gin"
    "github.com/golang-jwt/jwt/v4"
    "golang.org/x/time/rate"
)

type APIService struct {
    db          Database
    rateLimiter *rate.Limiter
    jwtSecret   []byte
}

// Middleware for JWT authentication
func (s *APIService) JWTAuthMiddleware() gin.HandlerFunc {
    return func(c *gin.Context) {
        tokenString := c.GetHeader("Authorization")
        if tokenString == "" {
            c.JSON(401, gin.H{"error": "Authorization header required"})
            c.Abort()
            return
        }
        
        token, err := jwt.Parse(tokenString, func(token *jwt.Token) (interface{}, error) {
            return s.jwtSecret, nil
        })
        
        if err != nil || !token.Valid {
            c.JSON(401, gin.H{"error": "Invalid token"})
            c.Abort()
            return
        }
        
        c.Set("user", token.Claims)
        c.Next()
    }
}

// Rate limiting middleware
func (s *APIService) RateLimitMiddleware() gin.HandlerFunc {
    return func(c *gin.Context) {
        if !s.rateLimiter.Allow() {
            c.JSON(429, gin.H{
                "error": "Rate limit exceeded",
                "retry_after": "60s",
            })
            c.Abort()
            return
        }
        c.Next()
    }
}

// RESTful endpoint with validation and error handling
func (s *APIService) CreateUser(c *gin.Context) {
    var user User
    if err := c.ShouldBindJSON(&user); err != nil {
        c.JSON(400, gin.H{"error": "Invalid request format"})
        return
    }
    
    // Validate input
    if err := user.Validate(); err != nil {
        c.JSON(400, gin.H{"error": err.Error()})
        return
    }
    
    // Create user in database
    createdUser, err := s.db.CreateUser(user)
    if err != nil {
        c.JSON(500, gin.H{"error": "Failed to create user"})
        return
    }
    
    c.JSON(201, gin.H{"data": createdUser})
}

// GORM database integration with connection pooling
package database

import (
    "gorm.io/gorm"
    "gorm.io/driver/postgres"
    "time"
)

type Database struct {
    DB *gorm.DB
}

func NewDatabase(dsn string) (*Database, error) {
    db, err := gorm.Open(postgres.Open(dsn), &gorm.Config{
        // Enable connection pooling
        ConnPool: &gorm.ConnPool{
            MaxIdleConns:    10,
            MaxOpenConns:    100,
            ConnMaxLifetime: time.Hour,
        },
    })
    
    if err != nil {
        return nil, err
    }
    
    // Auto-migrate schemas
    db.AutoMigrate(&User{}, &Role{}, &Permission{})
    
    return &Database{DB: db}, nil
}

// Optimized query with preloading and caching
func (d *Database) GetUserWithRoles(userID uint) (*User, error) {
    var user User
    err := d.DB.Preload("Roles.Permissions").
        Where("id = ?", userID).
        First(&user).Error
    
    if err != nil {
        return nil, err
    }
    
    return &user, nil
}

// Transaction example for complex operations
func (d *Database) CreateUserWithRoles(user *User, roleIDs []uint) error {
    return d.DB.Transaction(func(tx *gorm.DB) error {
        // Create user
        if err := tx.Create(user).Error; err != nil {
            return err
        }
        
        // Assign roles
        var roles []Role
        if err := tx.Find(&roles, roleIDs).Error; err != nil {
            return err
        }
        
        return tx.Model(user).Association("Roles").Append(roles)
    })
}
