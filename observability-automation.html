Observability Infrastructure Automation (Terraform & Go Controller)

Company: ThoughtSpot

Dates: Nov 2021 - April 2024

Designed and implemented cross-cloud Terraform infrastructure for observability, supporting AWS and GCP environments. Built a Terraform controller in Go and integrated it with ArgoCD for Kubernetes data plane automation.

Key Technologies & Tools

Terraform, Go, Kubernetes, ArgoCD, AWS, GCP, GitOps, Custom Resource Definitions (CRDs), client-go.

Situation

ThoughtSpot operated a complex, multi-cloud environment, with critical services deployed across both AWS and GCP. The observability infrastructure (including monitoring, alerting, and logging components) for these environments was often provisioned and managed manually or through disparate, non-standardized scripts. This fragmented approach led to significant inconsistencies in configurations, frequent configuration drift, increased human errors during setup, and substantial delays in bringing new Kubernetes data planes online with proper monitoring capabilities. The lack of automation was a major bottleneck for scaling operations.

Task

Automate the end-to-end deployment and ongoing management of observability infrastructure across both AWS and GCP using Infrastructure as Code (Terraform). Furthermore, the task involved building a Kubernetes-native controller in Go and integrating it with ArgoCD to enable GitOps-driven automation for provisioning observability components on new Kubernetes data planes.

Action

  • Meticulously authored modular, reusable, and cloud-agnostic Terraform modules for common observability components (Prometheus, Grafana, Fluent Bit, New Relic agents) adaptable to both AWS and GCP.
  • Designed and developed a custom Kubernetes controller from scratch in Go, including defining Custom Resource Definitions (CRDs) and implementing controller logic with the Kubernetes client-go library.
  • Integrated the controller to interact directly with the Terraform CLI, dynamically generating and executing Terraform configurations upon detecting changes in ObservabilityConfig CRDs.
  • Implemented robust mechanisms for managing Terraform state securely and comprehensive error reporting.
  • Configured ArgoCD to manage the deployment of the custom Go Terraform controller itself, ensuring its lifecycle was managed declaratively.
  • Designed the controller to be triggered by ArgoCD events or Git repository changes, ensuring new Kubernetes clusters automatically had their observability infrastructure set up.
  • Established and maintained robust CI/CD pipelines (Jenkins and GitHub Actions) to lint/validate Terraform, build/test the Go controller, and deploy it.
  • Conducted extensive testing, including unit tests, integration tests, and end-to-end tests, to ensure reliability and correctness of the automation.

Result

  • Successfully automated the end-to-end deployment and ongoing management of observability infrastructure across ThoughtSpot's complex multi-cloud footprint.
  • The Go Terraform controller, tightly integrated with ArgoCD, enabled a true GitOps approach for observability.
  • Drastically reduced manual effort by over 70%.
  • Improved configuration consistency by 90%.
  • Accelerated the onboarding of new Kubernetes data planes by 50% with pre-configured monitoring and logging.
  • Led to a more reliable and efficient operational environment.

Review

This initiative represented a significant leap forward in ThoughtSpot's infrastructure automation maturity and operational efficiency. My strong command of Terraform for Infrastructure as Code, deep expertise in Go for Kubernetes controller development, and a clear understanding of GitOps principles were instrumental in the project's success. The solution not only solved immediate pain points related to manual configuration and inconsistency but also laid a scalable and resilient foundation for future observability needs across diverse cloud environments. The technical depth required to build and integrate such a system was considerable, and its successful implementation demonstrated strong system design, automation, and problem-solving skills.
