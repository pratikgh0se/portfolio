Decentralized Feature Flag System with LaunchDarkly

Company: Highspot

Dates: April 2024 - Present

Designed, developed, and implemented a robust, decentralized feature flag system using LaunchDarkly across multiple microservices. This initiative aimed to enable controlled feature rollouts, A/B testing, and dynamic configuration updates, significantly reducing deployment risk and solving rigid deployment cycles.

Key Technologies & Tools

Clojure, Java, LaunchDarkly SDK, AWS Secrets Manager, External Secrets Operator (ESO), Kubernetes, ArgoCD, New Relic.

Situation

Highspot's existing deployment process was monolithic and rigid, characterized by infrequent, large-batch releases. This made it challenging to introduce new features incrementally, conduct A/B tests on user segments, or dynamically alter application behavior (e.g., enabling/disabling features for specific customers or environments) without requiring full, high-risk deployments. Such rigidity significantly slowed down iteration cycles, increased the potential for production incidents due to large blast radii, and limited the ability to perform dark launches or controlled rollouts.

Task

Design, develop, and integrate a flexible, secure, and highly available feature flag system using LaunchDarkly into the existing Clojure microservices architecture. The primary goal was to enable controlled feature rollouts, dynamic configuration management, and A/B testing capabilities, thereby reducing deployment risk and accelerating the pace of innovation.

Action

  • Spearheaded the architectural design of the new feature flag system, selecting LaunchDarkly as the preferred solution and writing core Clojure code to initialize the LaunchDarkly SDK client within the critical Magma microservice.
  • Developed a feature-service wrapper around the raw LaunchDarkly client to promote reusability and provide a simplified, consistent interface for other application components.
  • Implemented a sophisticated feature manager component responsible for coordinating between the feature-service and various flag sources, including a fallback mechanism to Mongo for legacy flags or in scenarios where LaunchDarkly might be unavailable.
  • Authored and championed a Pull Request (PR) for integrating the LaunchDarkly SDK key via External Secrets Operator (ESO) with AWS Secrets Manager, ensuring sensitive API keys were never hardcoded and securely distributed to Kubernetes pods.
  • Refactored the singleton initialization of the LaunchDarkly client from lazy loading to an eager initialization model within systems.clj and migrated lifecycle management to defstate for robust and predictable state management.
  • Developed custom scripts and comprehensive unit/integration tests to rigorously validate the end-to-end flag flow in Magma, including creating test cases for various scenarios and debugging complex runtime errors.
  • Implemented multi-context feature flag fetching to support sophisticated targeting rules.
  • Deployed the system to su0 (a specific scale unit within the latest environment) for live testing with real traffic.
  • Initiated the setup for data source monitoring of LaunchDarkly connectivity within New Relic, establishing alerts for service health.
  • Maintained active and transparent collaboration with team leads (Anna, Dylan) for architectural discussions, design reviews, and code feedback.
  • Partnered closely with other engineers (Pankaj, Justin) to debug complex inter-service issues.
  • Documented the entire integration process in Confluence and conducted formal knowledge transfer sessions for relevant engineering teams.

Result

  • Successfully designed, developed, and integrated a robust, decentralized feature flag system that is production-ready and deployed to su0.
  • Significantly reduced deployment risk by enabling controlled feature rollouts, dark launches, and A/B testing without requiring full application redeployments.
  • Led to faster iteration cycles for product development and improved developer productivity by decoupling feature releases from code deployments.
  • Enhanced the ability to conduct real-time experimentation and ensured secure key management through ESO integration, bolstering the platform's security posture.

Review

This project was a critical success, fundamentally transforming Highspot's feature delivery capabilities and accelerating its product development lifecycle. My ability to lead the design, implement complex Clojure code, integrate securely with external services, and conduct thorough, multi-faceted testing was paramount. The collaborative approach, especially in debugging intricate cross-service issues, proved highly effective in overcoming technical hurdles. The system now provides a flexible, resilient, and safe mechanism for future product innovation and experimentation, allowing Highspot to be more agile and responsive to market demands.
