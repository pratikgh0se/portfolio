Multi-Cloud Kubernetes Migration & Core Service Optimization

Company: Highspot

Dates: April 2024 - Present

Directed the migration and optimization of core services across various Kubernetes environments, including Amazon EKS and conceptualizing patterns for Azure AKS, enhancing scalability, reliability, and reducing operational overhead through strategic platform transitions.

Key Technologies & Tools

Amazon EKS, Azure AKS (conceptual), Kubernetes, Docker, Terraform, ArgoCD, AWS IAM, Prometheus, New Relic, External Secrets Operator (ESO).

Challenge

Highspot's core services were deployed across a mix of disparate and legacy infrastructure, including various Kubernetes platforms and older container orchestration systems. This created significant limitations in achieving consistent dynamic scalability, ensuring uniform reliability, and incurred high operational overhead due to fragmented management and maintenance across different cloud providers (e.g., AWS EKS, Azure AKS). The lack of a unified, modern approach hindered rapid development and efficient resource utilization, creating bottlenecks for growth in a multi-cloud landscape.

Action

  • Developed meticulous migration strategies for transitioning services between different Kubernetes platforms (e.g., to Amazon EKS, and conceptualizing/planning for Azure AKS integrations).
  • Utilized Terraform to provision and manage Kubernetes clusters (EKS, and defining patterns for AKS) in a declarative manner, focusing on cluster setup, worker node groups, and core service configurations.
  • Collaborated closely with various development teams to containerize and optimize core services for various Kubernetes environments.
  • Integrated deployment processes into existing CI/CD pipelines (e.g., using Buildkite for builds and tests, and ArgoCD for declarative Kubernetes deployments) for consistent and automated rollouts across different K8s platforms.
  • Implemented stringent security configurations within Kubernetes clusters, including fine-grained AWS IAM roles for Kubernetes service accounts (IRSA for EKS) and integrating with External Secrets Operator (ESO) for secure secret management across cloud boundaries.
  • Ensured seamless integration with unified observability stacks (Prometheus-Operator for metrics, New Relic for metrics, logs, and traces) to provide comprehensive, real-time monitoring of cluster health and application performance across multi-cloud K8s deployments.
  • Actively debugged complex deployment issues and optimized resource allocation (CPU and memory limits) for pods across diverse Kubernetes environments to achieve optimal performance, stability, and cost efficiency.

Result

  • Successfully directed the migration and optimization of critical core services to modern Kubernetes platforms (like Amazon EKS), demonstrating capabilities in managing multi-cloud containerized environments.
  • This resulted in significantly enhanced scalability and improved reliability across diverse cloud footprints.
  • The transition reduced operational overhead by automating infrastructure provisioning and streamlining cluster management, leading to improved resource utilization and tangible cost efficiency.
  • The established multi-cloud Kubernetes foundation provides a resilient and agile base for future service development and innovation.

Review

This project demonstrated senior-level capabilities in architecture, planning, and execution within complex multi-cloud Kubernetes environments. My leadership in driving infrastructure transitions and optimizing core services across platforms like EKS (and understanding the patterns for AKS) was crucial. The project not only modernized Highspot's core services but also laid a robust foundation for future growth and operational excellence in a multi-cloud strategy.
